# Linux Log Management

Linux systems store operational and security logs primarily within:

```text
/var/log
```

Common logging services include:

- auditd
- rsyslog
- journald
- syslog-ng

## Types of Linux Logs

### System Logs

Record operating system activity.

Examples include:

- Kernel events
- Boot information
- Service activity

---

### Application Logs

Contain logs generated by installed applications.

Examples include:

- Web servers
- Databases
- Mail servers

---

### Security Logs

Track authentication and security-related activities.

Examples include:

- User logins
- Failed authentication attempts
- Privilege escalation
- Account modifications

# Linux Audit Analysis

The Linux Audit Framework provides detailed visibility into authentication events and system activity.

## Audit Summary

Used the following command to obtain an overall audit summary:

```bash
aureport
```

This report provides statistics including:

- Logins
- Failed logins
- Authentications
- Failed authentications
- User activity
- Configuration changes

---

## Failed Event Summary

Reviewed failed security events using:

```bash
aureport --failed
```

### Observed Results

| Event | Count |
|---------|------:|
| Failed Logins | 263 |
| Failed Authentications | 1540 |

---

## Successful Login Investigation

Reviewed successful authentication events using:

```bash
ausearch -m USER_LOGIN -sv yes -i
```

The output identified:

- Successful SSH logins
- Authenticated users
- Login timestamps
- Source addresses

---

## Failed Login Investigation

Investigated failed authentication attempts using:

```bash
ausearch -m USER_LOGIN -sv no -i
```

This provided:

- Failed login attempts
- Target usernames
- Source IP addresses
- Authentication failures

---

## User-Specific Log Analysis

Filtered authentication events using:

```bash
ausearch -m USER_LOGIN -sv no -i | grep ct=root | wc -l
```

Result:

```text
227 failed root login attempts
```

Similarly:

```bash
ausearch -m USER_LOGIN -sv no -i | grep ct=mike | wc -l
```

Result:

```text
4 failed login attempts
```

# Monitoring

Monitoring continuously observes systems in real time to identify operational or security issues before they escalate.

Monitoring helps with:

- System health
- Performance tracking
- Security monitoring
- Capacity management
- Threat detection
- Compliance

---

# Logging vs Monitoring

| Logging | Monitoring |
|----------|------------|
| Records historical events | Observes systems in real time |
| Used for investigations | Used for immediate detection |
| Supports auditing | Supports operational awareness |
| Passive data collection | Active continuous observation |

# SIEM Fundamentals

Security Information and Event Management (SIEM) platforms centralize security logs from multiple systems and perform automated analysis to identify suspicious activity.

SIEM solutions improve enterprise visibility by correlating security events across infrastructure.

---

# Core SIEM Capabilities

## Data Aggregation

Collects logs from:

- Servers
- Firewalls
- Network Devices
- Applications
- Databases
- Security Appliances

---

## Event Correlation

Identifies relationships between seemingly unrelated events to detect suspicious behavior.

---

## Alerting

Automatically generates alerts when predefined security conditions are met.

Examples include:

- Multiple failed logins
- Privilege escalation
- Malware detection
- Unauthorized access

---

## Threat Investigation

Supports forensic analysis by allowing analysts to search historical events and reconstruct attack timelines.

---

## Threat Intelligence

Many SIEM platforms integrate external threat intelligence feeds to improve detection capabilities.

---

## Automation

Modern SIEM platforms can automatically respond to incidents by:

- Blocking IP addresses
- Disabling compromised accounts
- Isolating hosts
- Triggering incident response workflows
