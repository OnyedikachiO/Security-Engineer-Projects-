# Governance, Risk, Compliance & Security Frameworks

A hands-on **Security Engineering** project focused on implementing governance, risk management, compliance, and information security frameworks used to build resilient enterprise cybersecurity programs. This project explores governance structures, security regulations, GRC implementation, privacy regulations, NIST publications, ISO/IEC 27001, and SOC 2 through practical security engineering scenarios.

---

# Project Overview

Cybersecurity extends far beyond deploying technical controls. Modern organizations rely on governance, risk management, regulatory compliance, and internationally recognized security frameworks to ensure that information assets remain protected while supporting business objectives.

In this project, I explored how Security Engineers help organizations establish governance programs, assess and manage cyber risk, comply with regulatory obligations, implement security frameworks, and continuously improve organizational security maturity.

The project also examines globally recognized standards including **GDPR, PCI DSS, NIST SP 800-53, NIST SP 800-63B, ISO/IEC 27001, and SOC 2**, providing practical insight into how organizations build and maintain enterprise security programs.

All activities were completed within a controlled Security Engineering laboratory environment using governance documentation, regulatory guidance, and security framework implementation exercises.

## Governance vs Regulation

Although often confused, governance and regulation serve different purposes.

| Governance | Regulation |
|------------|------------|
| Internal organizational direction | External legal requirement |
| Defines security strategy | Enforced by government or regulators |
| Business driven | Law driven |
| Flexible | Mandatory |

Understanding this distinction is fundamental to enterprise Security Engineering.

# Information Security Framework

Modern organizations organize security documentation into structured frameworks.

## Policies

High-level security expectations.

Examples:

- Password Policy
- Acceptable Use Policy
- Access Control Policy

---

## Standards

Mandatory technical requirements.

Examples:

- Password Complexity Standard
- Encryption Standard
- Secure Configuration Standard

---

## Guidelines

Recommended best practices.

Examples:

- Secure Coding Guidelines
- Remote Work Guidelines
- Security Awareness Guidance

---

## Procedures

Detailed step-by-step operational instructions.

Examples:

- Incident Response Procedure
- User Provisioning Procedure
- Vulnerability Management Procedure

---

## Baselines

Minimum security configurations required across systems.

Examples:

- Windows Security Baseline
- Linux Hardening Baseline
- Server Configuration Baseline
